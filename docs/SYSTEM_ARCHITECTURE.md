# System Architecture: Resident soft

เอกสารฉบับนี้อธิบายโครงสร้างระบบและการทำงานของ **Resident soft** 

## 1. โครงสร้างผู้ใช้งาน (User Roles)
ระบบถูกออกแบบมาให้รองรับผู้ใช้งาน 3 ระดับ:
1. **Admin (ผู้ดูแลหอพัก / เจ้าของ):** มีสิทธิ์สูงสุดในการดู Dashboard รายได้, สร้างบิล, อนุมัติการชำระเงิน, จัดการสัญญาเช่า
2. **Staff (ช่างซ่อม / แม่บ้าน):** เข้าถึงเฉพาะส่วนงานแจ้งซ่อม (Maintenance) เพื่อรับงานและอัปเดตสถานะ (กำลังดำเนินการ, เสร็จสิ้น)
3. **Resident (ผู้เช่า / ลูกบ้าน):** เข้าสู่ระบบมาเพื่อดูใบแจ้งหนี้ประจำเดือน, แนบสลิปโอนเงิน, และกดแจ้งซ่อมของห้องตัวเอง

## 2. โมดูลการทำงานหลัก (Main Modules)

### 2.1 Room & Tenant Management (จัดการห้องพักและผู้เช่า)
- แสดงผังห้องพัก (Room Grid/List) แบ่งตามชั้น
- สถานะห้อง: ว่าง (Available), มีผู้เช่า (Occupied), กำลังปรับปรุง (Maintenance)
- การทำสัญญาเช่า: เมื่อมีผู้เช่าใหม่เข้ามา จะผูกผู้เช่าเข้ากับห้องพัก กำหนดวันเริ่มสัญญา และค่าเช่า/ค่ามัดจำ

### 2.2 Metering & Invoicing (จดมิเตอร์และออกบิล)
- ทุกๆ สิ้นเดือน Admin จะเข้าสู่หน้า **Meter Reading** เพื่อกรอกตัวเลขมิเตอร์น้ำ/ไฟ ของแต่ละห้อง
- ระบบจะคำนวณ: `(มิเตอร์เดือนนี้ - มิเตอร์เดือนที่แล้ว) * ราคาต่อหน่วย`
- รวมกับค่าเช่าห้องและค่าส่วนกลาง เพื่อสร้าง **Invoice (ใบแจ้งหนี้)**

### 2.3 Payment System (ระบบชำระเงิน)
- ลูกบ้านเห็น Invoice ใน Dashboard ของตนเอง
- ลูกบ้านสามารถอัปโหลดรูปภาพสลิปโอนเงิน (Payment Slip)
- Admin เห็นแจ้งเตือนว่ามีการชำระเงินใหม่ และสามารถกด **Approve (ยืนยัน)** เพื่อเปลี่ยนสถานะบิลเป็น "จ่ายแล้ว (Paid)"

### 2.4 Maintenance Requests (ระบบแจ้งซ่อม)
- ลูกบ้านเลือกหมวดหมู่ปัญหา (แอร์, น้ำ, ไฟฟ้า ฯลฯ) พร้อมพิมพ์รายละเอียดและแนบรูปถ่าย
- Staff และ Admin จะเห็นรายการแจ้งซ่อมและสามารถรับงานไปจัดการได้
