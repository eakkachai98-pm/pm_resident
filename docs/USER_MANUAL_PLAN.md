# User Manual & Onboarding Plan (Resident Soft)

เอกสารนี้ระบุแผนการสร้างคู่มือการใช้งาน (User Manual) และระบบแนะนำผู้ใช้ (Onboarding) โดยแบ่งเป็นเฟสจากง่ายไปยาก เพื่อให้สอดคล้องกับระยะเวลาในการ Launch ระบบ

## Phase 1: Quick Wins (ก่อน Launch ระบบ)
เน้นสิ่งที่ทำได้เร็ว ใช้เวลาพัฒนาน้อย และตอบโจทย์การใช้งานทันที

1. **คู่มือสำหรับพนักงาน (Staff / Admin Guide): PDF / Notion**
   - **วิธีทำ:** ใช้ Notion หรือ Google Docs แคปภาพหน้าจอด้วยมือเบื้องต้น และเขียนอธิบายทีละขั้นตอน (Step-by-step) 
   - **หัวข้อสำคัญ:** การเพิ่มข้อมูลลูกบ้าน, การรับเรื่องแจ้งซ่อม, การเปลี่ยนสถานะห้องพัก
   - **เหตุผล:** พนักงานมีจำนวนน้อย สามารถจัดอบรม (Training) ควบคู่กับการใช้ไฟล์ PDF ได้

2. **ระบบ Onboarding เบื้องต้นสำหรับลูกบ้าน (In-App Tooltips)**
   - **วิธีทำ:** ติดตั้ง Library เช่น `driver.js` เพื่อทำ Highlight ชี้จุดสำคัญเมื่อลูกบ้านล็อกอินครั้งแรก
   - **จุดที่ต้องชี้:**
     - 📌 เมนู "Report Issue" (แจ้งซ่อมตรงนี้นะ)
     - 📌 เมนู "My Profile & Documents" (ดูสัญญาและเอกสารตรงนี้นะ)
     - 📌 เมนูสลับภาษา (EN/TH/CN)

---

## Phase 2: Automation (ช่วง Launch / ปรับปรุงระบบ)
เน้นการลดภาระเมื่อ UI มีการเปลี่ยนแปลงในอนาคต

1. **Automated Screenshots (Playwright)**
   - **วิธีทำ:** เขียนสคริปต์ E2E Testing ด้วย Playwright
   - **หน้าที่ของสคริปต์:** สั่งให้บอทเปิดเบราว์เซอร์ ล็อกอินด้วยสิทธิ์ต่างๆ สลับภาษา (ไทย, จีน, อังกฤษ) แล้วแคปหน้าจอ (Screenshot) อัตโนมัติไปเก็บไว้ในโฟลเดอร์ `docs/assets/`
   - **เหตุผล:** เพื่อให้เรามีภาพประกอบคู่มือที่ "สดใหม่" และตรงกับ UI ปัจจุบันเสมอ โดยไม่ต้องเหนื่อยแคปเอง 3 ภาษา

---

## Phase 3: Full Knowledge Base (Post-Launch)
เมื่อระบบนิ่งและต้องการความน่าเชื่อถือสูงสุด

1. **Documentation Website (VitePress / Docusaurus)**
   - **วิธีทำ:** สร้างเว็บคู่มือแยกออกมา (เช่น `docs.resident-soft.com`) โดยใช้ VitePress 
   - **ฟีเจอร์:**
     - มีช่อง Search ค้นหาวิธีแก้ปัญหา
     - รองรับหลายภาษา (i18n)
     - ดึงรูปภาพอัตโนมัติจาก Phase 2 มาแสดง
   - **เหตุผล:** ลดภาระการตอบคำถามของ Admin (ลด Support Tickets) ให้ลูกบ้านสามารถค้นหาวิธีการใช้งานได้ด้วยตัวเอง

## ✅ ฟีเจอร์ที่พัฒนาแล้วเสร็จ (Completed Features)
1. **ระบบลายเซ็นลูกบ้าน (Digital Signature from Resident)**
   - เพิ่มระบบให้ลูกบ้านเซ็นชื่อผ่านหน้าจอ (Canvas) ของช่างบนมือถือ/แท็บเล็ต ตอนที่ช่างทำการปิดงาน (Resolve Ticket) เพื่อเป็นหลักฐานว่างานเสร็จสิ้น
   - เก็บข้อมูลรูปลายเซ็น (Base64) ลงใน Database Schema `MaintenanceTicket`
   - แสดงผลรูปลายเซ็นและรูปถ่ายการซ่อม (Repair Image) ในหน้าประวัติการซ่อม (History Modal) ของช่าง
